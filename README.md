# AI Fitness Coach

Your personal AI-powered fitness and nutrition coach — generate workout plans, analyze meal photos, check exercise form, and build daily habits, all powered by Google's Gemini AI.

**Live demo:** [https://ai-fitness-coach-ivory-three.vercel.app/]

---

## Features

### 🏋️ AI Workout Generator
Generates a personalized training plan based on your goal, experience level, available equipment, session length, and any injuries or limitations. The plan follows real programming principles — split selection based on training frequency (Full Body / Upper-Lower / Push-Pull-Legs), volume landmarks scaled to experience level, compound-before-isolation exercise ordering, and RPE/rest-period guidance for every exercise.

### 🥗 AI Meal Analyzer
Upload a photo of any meal and Gemini's vision model identifies the food and estimates calories and macros (protein, carbs, fat). Results are fully editable before saving, since AI estimation is approximate — the user stays in control of what gets logged. Daily history and running calorie totals are tracked automatically.

### 📸 AI Form Checker
Upload a short video clip of an exercise and receive specific, actionable coaching feedback — what's being done well, what to correct, and any safety concerns — generated by Gemini's video understanding capabilities.

### 🔥 Habit Tracker
Simple daily habit checklist with automatic streak calculation.

### 📊 Progress Dashboard
Visual overview of the last 7 days: calorie intake by day, saved workout and form-check counts, and a habit-consistency grid.

## Architecture Notes

- **Auth & data isolation:** every table (`meals`, `workouts`, `habits`, `form_checks`) uses Supabase Row Level Security, so a user can only ever read or write their own rows — enforced at the database level, not just in application code.
- **AI as a proposal, not a verdict:** meal analysis results are editable before being saved, since vision-based calorie estimation is inherently approximate. The UI treats AI output as a first draft the user can correct, not a final answer.
- **Structured AI output:** all Gemini calls are prompted to return strict JSON matching a defined schema, parsed server-side in Next.js API routes — this keeps the AI's non-deterministic output contained behind a predictable contract for the frontend.
- **Storage security:** photo and video uploads go into per-user folders (`{user_id}/filename`) within Supabase Storage, with insert policies that verify the folder matches the authenticated user's ID.

## Getting Started Locally

### Prerequisites
- Node.js 20+
- A free [Supabase](https://supabase.com) project
- A free [Google AI Studio](https://aistudio.google.com) API key

### Setup

```bash
git clone https://github.com/YOUR_USERNAME/ai-fitness-coach.git
cd ai-fitness-coach
npm install
```

Create a `.env.local` file in the project root:

```
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_publishable_key
GEMINI_API_KEY=your_gemini_api_key
```

Set up the database schema by running the SQL in `/supabase/schema.sql` (see below) in your Supabase project's SQL Editor, then create two public storage buckets: `meal-photos` and `form-videos`, each with insert/select policies scoped to the authenticated user's folder.

Run the dev server:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## What I Learned Building This

This was my first full-stack AI application, built from scratch. Some of the real-world engineering issues I worked through:

- Debugging a misplaced API route file causing silent 404s
- Handling a deprecated AI model (`gemini-2.0-flash` → `gemini-2.5-flash`) and diagnosing quota errors across multiple Google Cloud projects

- Timezone-aware timestamp handling (`timestamp` vs `timestamptz` in PostgreSQL)
